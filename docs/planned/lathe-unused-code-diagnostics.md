# Lathe — Unused Code Diagnostics

## Goal

Provide real-time editor feedback for unused code elements:
* Unused private methods
* Unused private fields
* Unused local variables

These elements will be highlighted to the user as faded-out code in their editor without triggering noise in compile error lists.

---

## LSP Presentation

Unused declarations will be reported as part of the normal compiler diagnostic loop.
Lathe will publish them with:

* **Severity:** `DiagnosticSeverity.Hint` (ensures they do not clutter warning/error lists or block builds).
* **Tags:** `List.of(DiagnosticTag.Unnecessary)` (instructs LSP clients like Neovim and VS Code to fade out the unused range).
* **Source:** `"lathe"`

---

## Detection Logic

Diagnostics will be generated by running a custom AST scanner (`UnusedDeclarationScanner`) over the attributed tree (`AttributedFileAnalysis`) compiled during fast-pass or full compilation.

### Unused Private Methods

1. **Candidates:**
   Identify all method declarations (`MethodTree`) containing the `Modifier.PRIVATE` flag.
   
2. **Exclusions:**
   * Private parameterless constructors (often used in static utility classes to prevent instantiation).
   * Serialization methods called reflectively by the JVM (e.g., `readObject`, `writeObject`, `readResolve`, `writeReplace`).
   * Methods annotated with framework-specific annotations (e.g., if any annotation is present that could trigger reflection, though we can start conservative with simple private methods).

3. **Call Graph & Reachability:**
   To handle self-recursion and cyclic dependencies (e.g., private method `A` calls `B`, and `B` calls `A`, but neither is accessed from public code), we perform a reachability analysis:
   * **Roots:** Any non-private methods, non-private fields, constructors, static initializers, and instance initializers.
   * **Edges:** Traverse the compilation unit to find all method call sites (`MethodInvocationTree`) and method references (`MemberReferenceTree`). Resolve the target element using `Trees.getElement(TreePath)`.
   * **Check:** Any candidate private method not reachable from the roots is marked unused.

### Unused Private Fields

1. **Candidates:**
   Identify all variable declarations (`VariableTree`) declared directly under a class declaration that carry the `Modifier.PRIVATE` flag.
   
2. **Exclusions:**
   * Fields representing special JVM parameters (e.g., `serialVersionUID`).

3. **Tracking:**
   * Traverse all simple-name identifiers (`IdentifierTree`) and member selectors (`MemberSelectTree`).
   * Resolve their target elements.
   * A private field is marked unused if it has no read references anywhere in the class.
   * *(Note: We should ignore references that are purely write-only assignments if the value is never read, though a simple first implementation can check for any references outside of the field declaration).*

### Unused Local Variables

1. **Candidates:**
   Identify all variable declarations (`VariableTree`) inside method blocks or initializers.
   This excludes class-level fields and method parameters.

2. **Tracking:**
   * Scan the parent method or initializer block for any references (`IdentifierTree`) resolving to the local variable.
   * If there are no references to the variable's element other than its declaration, the variable is marked unused.

---

## Pipeline Integration

1. After the compilation finishes successfully and generates an `AttributedFileAnalysis` in `SourceAnalysisSession.compile(...)`, we invoke:
   ```java
   List<Diagnostic> unusedDiags = UnusedDeclarationScanner.scan(analysis);
   ```
2. Merge `unusedDiags` with the compiled diagnostic output before returning.
3. Avoid running this check if compilation yields severe parser errors that prevent constructing a reliable attributed AST.

---

## Tests

JUnit tests should verify:

* Unused private methods are correctly identified.
* Recursive private methods that call themselves but have no external callers are flagged.
* Mutual recursion/cycles of unused private methods are flagged.
* Unused private fields are flagged.
* Unused local variables are flagged.
* Unused elements inside nested/inner classes are flagged.
* Public/protected/package-private elements are **not** flagged.
* Private utility constructors are **not** flagged.
* SerialVersionUID is **not** flagged.
